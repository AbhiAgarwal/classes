Scala Development Tools

The current version of Scala is 2.11.7 at the start of Fall 2015. We standardize to this version.

Scala IDE Import

To import the project files into the Scala IDE, go to

File > Import > Existing Projects into Workspace (under General)

and then select

Select archive file:

and browse to the file popl.zip that you downloaded from the course website.

Running Applications from inside the Scala IDE

After you have have imported the project, you can run Scala applications that are compiled from the source files in the project. To do so, right-click in the package explorer on the source file that defines the application's entry point and choose

Run As > 1 Scala Application

For example, run the file src/main/scala/class1/ScalaGreeter.scala as a Scala application and you will see the message Hello Scala printed in the console.

You can run Java applications in a similar manner. For example, run the file src/main/java/class1/JavaGreeter.scala as a Java application and you will see the message Hello Scala from Java printed in the console.

Command-Line Tools

While not required if you have installed Scala IDE, you may also want to install sbt (follow the installation instructions provided in the notes for class 2).

For your convenience, an sbt script (build.sbt) is included. To use sbt, open a terminal window and change to the directory containing the workspace of your Scala IDE. After you have imported popl.zip, you will find a directory named popl in your workspace directory. Change into that directory. Now you can issue the following commands to compile and run the code in the project by executing

$ sbt run

The source files in the popl project provide multiple application entry points. sbt's run command will detect these automatically and prompt you to choose one of them.

It is often convenient to run sbt interactively. Type

$ sbt

and then run via

> run

at the sbt prompt.
